# Requirements: Энигма

**Defined:** 2026-03-19
**Core Value:** Безопасная отправка корпоративных табличных данных в облачные LLM без утечки чувствительной информации

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Загрузка файлов

- [x] **LOAD-01**: Пользователь может загрузить Excel-файл (xlsx) с несколькими листами
- [x] **LOAD-02**: Пользователь может загрузить CSV-файл
- [x] **LOAD-03**: Система сохраняет структуру файла (все листы, колонки, порядок строк)

### Детекция чувствительных данных

- [ ] **DETC-01**: Система автоматически предлагает чувствительные колонки на основе эвристик по названиям
- [ ] **DETC-02**: Пользователь может добавить/убрать колонки для маскирования через чекбоксы
- [ ] **DETC-03**: Для каждой выбранной колонки система определяет тип маскирования (текст или число)

### Маскирование

- [ ] **MASK-01**: Текстовые значения заменяются на псевдонимы с авто-префиксом из названия колонки (напр. "Предприятие A")
- [ ] **MASK-02**: Числовые значения умножаются на случайный коэффициент (0.5–1.5) с сохранением пропорций
- [ ] **MASK-03**: Одно уникальное значение = один псевдоним во всём файле (кросс-листовая консистентность)
- [ ] **MASK-04**: Идентификаторы-числа (номера договоров, документов) маскируются как текст, а не умножением

### Выгрузка

- [ ] **OUT-01**: Пользователь скачивает замаскированный Excel-файл со всеми листами
- [ ] **OUT-02**: Пользователь скачивает маппинг в формате JSON
- [ ] **OUT-03**: Пользователь скачивает маппинг в формате Excel
- [ ] **OUT-04**: Система показывает статистику: количество замаскированных значений и уникальных сущностей

### Дешифровка

- [ ] **DECR-01**: Пользователь загружает замаскированный файл + JSON-маппинг
- [ ] **DECR-02**: Система заменяет псевдонимы обратно на оригинальные значения
- [ ] **DECR-03**: Новые колонки/строки (добавленные LLM) остаются без изменений

### Интерфейс

- [ ] **UI-01**: Интерфейс полностью на русском языке
- [x] **UI-02**: Stateless — сервер ничего не хранит, данные только в сессии Streamlit

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Расширенная детекция

- **DETC-04**: Детекция чувствительных колонок по жёлтой заливке ячеек (openpyxl fill color)
- **DETC-05**: Распознавание русских сущностей по значениям (ООО/ОАО/ИП regex, ФИО паттерны)

### Расширенная дешифровка

- **DECR-04**: Нечёткая дешифровка (fuzzy matching) если LLM слегка изменил написание псевдонима

### Новые форматы

- **FMT-01**: Маскирование Word-документов (.docx)
- **FMT-02**: Маскирование PowerPoint-презентаций (.pptx)

### Многопользовательность

- **AUTH-01**: Авторизация / аккаунты пользователей

## Out of Scope

| Feature | Reason |
|---------|--------|
| Хранение маппингов на сервере | Противоречит stateless архитектуре и принципу безопасности |
| Интеграция с LLM API | Scope creep, отдельный продукт |
| Переключаемый язык UI | Только русский в v1, целевая аудитория русскоязычная |
| Формальные модели приватности (k-anonymity) | Несовместимо с задачей анализа данных LLM |
| Облачный деплой с публичным URL | Риск безопасности без авторизации, деплой на корп. VM |
| Стриминговая обработка | Батчевая обработка файлов достаточна для v1 |
