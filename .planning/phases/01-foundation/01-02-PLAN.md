**Step 4 — Create `ui/step_indicator.py`:**

Render a horizontal step indicator showing 3 steps for the masking flow. Current step is highlighted.

```python
import streamlit as st

STEPS = ["Загрузка файла", "Выбор колонок", "Результат"]

def render_steps(current: int) -> None:
    """Render step indicator. current is 1-based (1, 2, or 3)."""
    cols = st.columns(len(STEPS))
    for i, (col, label) in enumerate(zip(cols, STEPS), start=1):
        if i == current:
            col.markdown(f"**:blue[Шаг {i}: {label}]**")
        elif i < current:
            col.markdown(f"~~Шаг {i}: {label}~~")
        else:
            col.markdown(f"Шаг {i}: {label}")
```

Use `st.columns` for horizontal layout. Current step: bold blue text. Completed steps: strikethrough. Future steps: plain text. No custom HTML — pure Streamlit markdown.

**Step 5 — Create `ui/upload_widget.py`:**

```python
import streamlit as st
import pandas as pd

def render_preview(sheets: dict[str, pd.DataFrame]) -> None:
    """Render preview of parsed sheets. 20 rows per sheet.
    Single sheet: no tabs, just dataframe.
    Multiple sheets: st.tabs with sheet names.
    """
    sheet_names = list(sheets.keys())
    if len(sheet_names) == 1:
        st.dataframe(
            sheets[sheet_names[0]].head(20),
            use_container_width=True,
        )
    else:
        tabs = st.tabs(sheet_names)
        for tab, name in zip(tabs, sheet_names):
            with tab:
                st.dataframe(
                    sheets[name].head(20),
                    use_container_width=True,
                )
```

Key: `.head(20)` for exactly 20 preview rows. `use_container_width=True` for wide layout. Single sheet = no tabs (per Claude's discretion from CONTEXT.md).

**Step 6 — Create `pages/masking.py`:**

This is the masking flow page. Phase 1 implements only Step 1 (upload + preview). Steps 2 and 3 will be added in Phase 2 and Phase 3.

```python
import streamlit as st
from core.state_keys import SHEETS, RAW_BYTES, STAGE, FILE_NAME, STAGE_UPLOADED
from core.parser import parse_upload
from ui.upload_widget import render_preview
from ui.step_indicator import render_steps


def render() -> None:
    st.header("Маскирование данных")

    stage = st.session_state.get(STAGE)

    if stage is None:
        _render_step_upload()
    elif stage == STAGE_UPLOADED:
        _render_step_preview()


def _render_step_upload() -> None:
    render_steps(current=1)
    st.subheader("Загрузите файл для маскирования")

    uploaded_file = st.file_uploader(
        "Выберите файл Excel (.xlsx) или CSV (.csv)",
        type=["xlsx", "csv"],
        key="file_uploader_mask",
    )

    if uploaded_file is not None:
        try:
            sheets = parse_upload(uploaded_file)
            st.session_state[SHEETS] = sheets
            st.session_state[FILE_NAME] = uploaded_file.name
            st.session_state[STAGE] = STAGE_UPLOADED
            st.rerun()
        except ValueError as e:
            st.error(str(e))


def _render_step_preview() -> None:
    render_steps(current=1)
    sheets = st.session_state[SHEETS]
    file_name = st.session_state.get(FILE_NAME, "файл")

    st.subheader(f"Превью: {file_name}")
    render_preview(sheets)

    col_back, col_next = st.columns([1, 1])
    with col_back:
        if st.button("Сбросить", use_container_width=True):
            for key in [SHEETS, RAW_BYTES, STAGE, FILE_NAME]:
                st.session_state.pop(key, None)
            st.rerun()
    with col_next:
        if st.button("Далее", type="primary", use_container_width=True, disabled=True):
            pass  # Phase 2 will enable this
```

Key decisions:
- After successful parse, set STAGE and call `st.rerun()` to transition to preview
- "Сбросить" (Reset) button clears all session state keys and reruns — user returns to upload
- "Далее" (Next) button is disabled — Phase 2 will enable it when column selection is implemented
- Step indicator always shows Step 1 (upload/preview are both part of Step 1 in this phase)
- File uploader uses `key="file_uploader_mask"` to distinguish from decryption flow
- Error messages bubble up from `parse_upload` as `ValueError` — displayed via `st.error`

**Step 8 — Create `app.py`:**

```python
import streamlit as st

st.set_page_config(
    page_title="Enigma — Шифрование данных для LLM",
    layout="wide",
    initial_sidebar_state="expanded",
)

st.sidebar.title("Enigma")
page = st.sidebar.radio(
    "Режим",
    ["Маскирование", "Дешифровка"],
    label_visibility="collapsed",
)

if page == "Маскирование":
    from pages.masking import render
    render()
elif page == "Дешифровка":
    from pages.decryption import render
    render()
```

Key: `st.set_page_config` MUST be the first Streamlit command. Sidebar has radio with two options. Dynamic import of page modules based on selection. No Streamlit native multi-page app (pages/ directory convention) — we use manual routing via sidebar radio to have full control over navigation.

**Important: Do NOT use Streamlit's built-in multi-page app pattern** (where files in `pages/` are auto-discovered). Instead, `app.py` is the single entry point and manually imports from `pages/masking.py` and `pages/decryption.py` based on sidebar selection. This gives us control over navigation labels and avoids Streamlit auto-generating sidebar links.
