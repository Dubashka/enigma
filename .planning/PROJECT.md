# Энигма — Маскирование данных для LLM

## What This Is

Streamlit-приложение для маскирования чувствительных данных в Excel/CSV файлах перед отправкой в облачные LLM. Пользователь загружает файл, система автоматически предлагает колонки для маскирования, заменяет чувствительные данные псевдонимами, и отдаёт замаскированный файл + файл маппинга для обратной дешифровки. Целевая аудитория — сотрудники промышленных компаний, работающие с закупками, ТОиР, логистикой.

## Core Value

Безопасная отправка корпоративных табличных данных в облачные LLM без утечки чувствительной информации (названия контрагентов, ФИО, суммы, номера договоров).

### Active

- [ ] Загрузка Excel (многолистовых) и CSV файлов
- [ ] Автоматическое определение чувствительных колонок с возможностью корректировки чекбоксами
- [ ] Маскирование текстовых данных: замена на псевдонимы с авто-префиксом из названия колонки (колонка "Имя предприятия" → "Предприятие A", "Предприятие B")
- [ ] Маскирование числовых данных: умножение на случайный коэффициент (0.5–1.5) с сохранением пропорций
- [ ] Консистентное маскирование: одно уникальное значение = один псевдоним в рамках всего файла (кросс-листово)
- [ ] Выгрузка замаскированного Excel-файла (с сохранением структуры листов)
- [ ] Выгрузка файла маппинга в двух форматах: JSON (для системы) и Excel (для пользователя)
- [ ] Обратная дешифровка: загрузка замаскированного файла + маппинг (JSON) → восстановление оригинальных значений
- [ ] Дешифровка работает как замена значений — новые колонки/строки от LLM остаются как есть
- [ ] Превью загруженного файла перед маскированием
- [ ] Статистика после маскирования (количество замаскированных значений, уникальных сущностей)
- [ ] Интерфейс полностью на русском языке
- [ ] Stateless архитектура — сервер ничего не хранит, данные только в файлах пользователя

### Out of Scope

- Авторизация/логин — v1 без аутентификации, решим позже
- Маскирование Word/PPT файлов — запланировано на v2+
- Хранение истории маскирований на сервере — пользователь хранит маппинги сам
- Переключаемый язык интерфейса — только русский в v1

## Context

- Данные — промышленные: закупки ТМЦ, сообщения о неисправностях (ТОиР), услуги подрядчиков, транспортировка сырья
- Типичные чувствительные поля: названия компаний-контрагентов (ООО "ЛУИС+", ООО "СТРОНГ-ФИЛЬТР"), ФИО сотрудников (Леонов Алексей, Фурсов Олег), суммы/цены, номера договоров, наименования цехов/агрегатов
- Excel-файлы могут содержать 4+ листов с 15–37 колонками и до десятков тысяч строк
- Уже есть VM для деплоя: 158.160.27.49 (SSH доступ)
- В рабочей папке есть пример файла: "Данные для маскирования_13.03.xlsx" с 4 листами и жёлтой подсветкой чувствительных колонок

## Constraints

- **Tech stack**: Streamlit + Python (openpyxl для Excel, pandas для обработки)
- **Deployment**: VM по SSH (158.160.27.49)
- **No server storage**: Все данные — в сессии Streamlit, ничего не пишем на диск кроме temp-файлов
- **Performance**: Должен обрабатывать файлы с десятками тысяч строк без зависания

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Stateless (2 файла вместо серверного хранения) | Простота, безопасность, нет нужды в БД | — Pending |
| Авто-префикс из названия колонки | Баланс между читаемостью и автоматизацией, LLM понимает тип сущности | — Pending |
| Коэффициент для чисел вместо рандомизации | Сохраняет пропорции между строками — важно для анализа данных LLM | — Pending |
| Единый маппинг кросс-листово | Позволяет LLM связывать данные между листами одного файла | — Pending |
| JSON + Excel для маппинга | JSON для обратной обработки, Excel для человека | — Pending |
