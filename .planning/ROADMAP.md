# Roadmap: Энигма

## Overview

Четыре фазы, от скелета приложения до деплоя на VM. Фаза 1 создаёт фундамент с загрузкой файлов и правильной архитектурой session state — это разблокирует всё остальное. Фаза 2 строит движок детекции и маскирования — ядро продукта. Фаза 3 добавляет выгрузку файлов и полный цикл дешифровки. Фаза 4 деплоит на VM и верифицирует производительность на реальных данных.

## Phases

- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

- [ ] **Phase 2: Detection + Masking** - Движок детекции чувствительных колонок и полный движок маскирования
- [ ] **Phase 3: Output + Decryption** - Выгрузка замаскированных файлов, маппингов, статистики и дешифровка
- [ ] **Phase 4: Deployment** - Деплой на VM (systemd + nginx), hardening производительности

## Phase Details

### Phase 1: Foundation
**Goal**: Пользователь может загрузить Excel/CSV файл, увидеть превью данных и начать работу в приложении с полностью русскоязычным интерфейсом и надёжной stateless архитектурой
**Depends on**: Nothing (first phase)
**Requirements**: LOAD-01, LOAD-02, LOAD-03, UI-01, UI-02
**Success Criteria** (what must be TRUE):
  1. Пользователь загружает xlsx-файл с несколькими листами и видит превью первых строк каждого листа
  2. Пользователь загружает CSV-файл и видит превью данных
  3. Все надписи, кнопки и сообщения в интерфейсе на русском языке
  4. Перезагрузка страницы сбрасывает сессию — никакие данные не сохраняются на сервере
  5. Структура файла (все листы, колонки, порядок строк) сохранена в session state и доступна следующим шагам
**Plans**: 2 plans
Plans:
- [ ] 01-01-PLAN.md — Project scaffolding, core parser with TDD, session state keys
- [ ] 01-02-PLAN.md — Streamlit UI: app entry point, sidebar, upload, preview, theme

### Phase 2: Detection + Masking
**Goal**: Пользователь может автоматически получить предложение чувствительных колонок, скорректировать его чекбоксами и запустить маскирование — результатом является полностью замаскированный файл с консистентным кросс-листовым маппингом
**Depends on**: Phase 1
**Requirements**: DETC-01, DETC-02, DETC-03, MASK-01, MASK-02, MASK-03, MASK-04
**Success Criteria** (what must be TRUE):
  1. После загрузки файла система автоматически предлагает список чувствительных колонок на основе названий
  2. Пользователь может добавить или убрать колонки через чекбоксы, и маскирование применяется только к отмеченным
  3. Текстовая колонка "Имя предприятия" маскируется в "Предприятие A", "Предприятие B" — псевдоним содержит слово из названия колонки
  4. Одно и то же значение (например, название контрагента) получает один и тот же псевдоним на всех листах файла
  5. Числовые значения в количественных колонках умножаются на коэффициент, числа-идентификаторы (номера договоров) маскируются как текст

### Phase 3: Output + Decryption
**Goal**: Пользователь может скачать замаскированный файл и файлы маппинга, увидеть статистику маскирования, а затем восстановить оригинальные данные из замаскированного файла
**Depends on**: Phase 2
**Requirements**: OUT-01, OUT-02, OUT-03, OUT-04, DECR-01, DECR-02, DECR-03
**Success Criteria** (what must be TRUE):
  1. Пользователь скачивает замаскированный Excel-файл с сохранёнными листами, заголовками и структурой
  2. Пользователь скачивает маппинг в JSON и в Excel — оба файла содержат полное соответствие псевдоним → оригинал
  3. После маскирования видна статистика: количество замаскированных значений и уникальных сущностей
  4. Пользователь загружает замаскированный файл + JSON-маппинг и скачивает файл с восстановленными оригинальными значениями
  5. Колонки и строки, добавленные LLM в замаскированный файл, остаются в восстановленном файле без изменений

### Phase 4: Deployment
**Goal**: Приложение запущено на VM и доступно по внутреннему URL, обрабатывает реальный файл "Данные для маскирования_13.03.xlsx" без зависания, данные не утекают между сессиями
**Depends on**: Phase 3
**Requirements**: (infrastructure — no dedicated v1 req-ids; satisfies production readiness implied by all requirements)
**Success Criteria** (what must be TRUE):
  1. Приложение доступно по адресу VM (158.160.27.49) через nginx и остаётся запущенным после disconnect SSH
  2. Файл с 4 листами и ~30 000 строк обрабатывается без зависания UI менее чем за 30 секунд
  3. Два параллельных браузерных сеанса не влияют друг на друга — данные одного пользователя не видны другому
  4. Загрузка файла размером >50 MB выдаёт читаемое сообщение об ошибке вместо краша
